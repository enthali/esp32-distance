Supporting Documentation Design Specifications¶

This document defines the required content for each traceable supporting documentation file. One SPEC element per chapter — specifying what each chapter must cover, not how it is written.

Document Version: 1.0 Last Updated: 2026-03-16

README.md¶

Design Specification: README Introduction and Project Link SPEC_DEV_README_INTRO
status: approved
tags: developer, documentation, readme
links outgoing: REQ_DEV_README_1, SPEC_ARCH_LAYERS_1

Description: The README introduction section SHALL contain a short project description (≤3 sentences) and a prominent link to the GitHub Pages documentation site.

Required content:

  • Project name and one-line purpose statement

  • What the device does (distance → LED feedback)

  • Link to full documentation on GitHub Pages

Explicitly excluded: Hardware specs, pin tables, build instructions, QEMU steps — these live in the Sphinx docs to avoid duplication.

Traceability note: If SPEC_ARCH_LAYERS_1 changes the component structure, the project description in README SHALL be reviewed.
Design Specification: README Metadata Section SPEC_DEV_README_META
status: approved
tags: developer, documentation, readme
links outgoing: REQ_DEV_README_1

Description: The README SHALL end with a metadata section covering project governance information.

Required content:

  • License statement with SPDX identifier

  • Contributing pointer (link to CONTRIBUTING.md or inline note)

Design Specification: Overview Index — Project Description SPEC_DEV_OV_INDEX_WHAT
status: approved
tags: developer, documentation, overview
links outgoing: REQ_DEV_OV_INDEX_1, US_DISPLAY_1

Description: The “What Is This Project?” section SHALL describe the device purpose in plain language for a first-time reader.

Required content:

  • Primary use case: parking aid

  • Maker / builder (hardware assembler)

  • Developer (firmware / documentation contributor)

Required content:

  • Links to key sections (overview, requirements, design, guides)

  • Brief description of what each section contains

docs/01_overview/hardware.rst¶

Design Specification: Hardware Components Listing SPEC_DEV_OV_HW_COMPONENTS
status: approved
tags: developer, documentation, hardware
links outgoing: REQ_DEV_OV_HW_1, REQ_SNS_1, REQ_LED_1

Description: The hardware components section SHALL list all required and optional components with specifications.

Required content:

  • ESP32 development board (type, flash size)

  • HC-SR04 ultrasonic sensor (operating voltage, range)

  • WS2812 LED strip (addressable, power requirements)

  • Power supply notes

Traceability note: SHALL be reviewed when REQ_SNS_1 or REQ_LED_1 change component specifications.

Required content:

  • GPIO assignment table (pin number, signal name, connected component)

  • Note on ESP32 strapping pins to avoid

  • Wiring diagram reference or inline wiring notes
